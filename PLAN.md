## 1. 项目概述

### 1.1 项目定位

这是一个**以"礼物"为核心定位**的个人小程序。使用者分为两类：

| 角色 | 行为 | 平台 |
|------|------|------|
| **我（发布者）** | 在电脑上发布日常动态、上传狗狗照片 | 微信云开发控制台 / 后期 H5 管理端 |
| **女朋友 & 家人（浏览者）** | 在手机上浏览内容、查看照片 | 微信小程序 |

### 1.2 核心功能

1. **首页** — 沉浸式封面大图 + 下滑探索，营造"翻开私人画册"的体验
2. **日常记录** — 图文动态时间线，记录生活中的点滴
3. **狗狗相册** — 独立的狗狗专题板块，拍立得瀑布流 + 全屏查看
4. **关于我们** — 温情介绍页

### 5.5 卡片设计

所有内容卡片遵循统一风格：

- 白色背景 + 16rpx 圆角
- 微阴影: `box-shadow: 0 4rpx 16rpx rgba(0,0,0,0.04)`
- 暖色边框: `border: 1rpx solid #F0EBE8`
- 点击态: `transform: scale(0.98)` + 过渡动画
- 卡片间距: 24rpx

### 5.6 动效规范

| 场景 | 动效 | 时长 |
|------|------|------|
| 页面进入 | 淡入 + 上移 (fade-in-up) | 300ms |
| 点击反馈 | 缩放 (scale 0.98) | 150ms |
| 图片加载 | 骨架屏 shimmer → 淡入 | 300ms |
| 全屏切换 | 弹性滑动 | 300ms |
| 滚动 | 默认原生滚动 | — |

CSS 动画使用 `transform` 和 `opacity` 属性以利用 GPU 加速。

```javascript

  value:        Any,           // 配置值
  updatedAt:    Date,
```

- `posts`: `{ createdAt: -1 }` — 按时间倒序查询动态

- `site_config`: `{ key: 1 }` — 按键查询

**交互细节**：

**实现要点**：
- 封面轮播可用 `swiper` 组件
- 入口卡片用 `position: absolute` / `sticky` 定位在封面下方
- 使用 uni-app 的 `onPullDownRefresh` 和 `onReachBottom` 实现下拉刷新和触底加载

**瀑布流实现方案**：
- 采用 **JS 双列动态分配算法**（兼容性最好）：
  1. 维护左右两列高度
  2. 每张新照片放入当前较矮的列
  3. 使用 `wx.createSelectorQuery()` 或 `uni.createSelectorQuery()` 获取实际列高
- 每张照片预计算显示高度：`displayHeight = (colWidth / photoWidth) * photoHeight`

**拍立得样式**：

```scss
.polaroid-item {
  background: #fff;
  padding: 12rpx 12rpx 40rpx 12rpx;  // 底部留白模拟拍立得
```

## 9. 部署策略

### 9.1 主方案：微信小程序测试版

```
开发流程:
  1. 注册微信小程序账号 (个人主体)
  2. 开通云开发环境 (免费额度)
  3. uni-app 开发，微信开发者工具调试
  4. 上传代码 → 设为体验版
  5. 添加女朋友和家人微信为体验者
  6. 他们通过体验版小程序访问

优势:
  - 微信生态内，体验流畅
  - 云开发免费额度充足
  - 测试版可无限期使用
  - 不需要审核发布
```

### 9.2 兜底方案：GitHub Pages (H5)

```
部署流程:
  1. 在 GitHub 创建 <username>.github.io 仓库
  2. uni-app 编译 H5: npm run build:h5
  3. 将 dist/build/h5 推送到仓库
  4. 访问 https://<username>.github.io 即可查看

优势:
  - 完全免费
  - HTTPS + CDN
  - 无需微信环境即可浏览

劣势:
  - 纯静态，无法动态发布内容
  - 需配合云开发数据库读取 (需配置安全域名)
```

### 9.3 两套部署的关系

```
                    uni-app 代码
                         │
            ┌────────────┼────────────┐
            ▼            ▼            ▼
      微信小程序      H5 (GitHub)    管理端 H5
      (主战场)       (兜底预览)      (后期)
            │            │
            └─────┬──────┘
                  ▼
            微信云开发
           (数据库 + 存储)
```

两套前端共用同一套云开发后端数据。
